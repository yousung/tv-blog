# tv.lovizu.com 콘텐츠 전략 및 프로그램 우선순위

작성일: 2026-06-16
담당: CMO
연결 이슈: BLOA-312

## 목표

`tv.lovizu.com`은 방송에 소개된 맛집, 지역, 생활·건강 정보를 독자가 방문 전 확인할 수 있는 형태로 정리하는 별도 블로그로 운영한다. 초기 목표는 자동 발행이 아니라, 검증 가능한 초안 생산 흐름과 프로그램별 우선순위를 확정해 반복 가능한 콘텐츠 파이프라인을 만드는 것이다.

## 핵심 KPI

- 2주: 초안 10건 이상, 발행 가능 품질 통과 5건 이상
- 4주: Google 색인 확인 글 5건 이상
- 6주: 프로그램명+지역+상호 또는 메뉴 롱테일 키워드 노출 발생
- 품질: 발행 글 100%에 방송사 출처, 지도/공식 채널 확인, 방문 전 확인 문구 포함
- 안전: 건강 글 100%에 일반 정보 고지와 의학적 단정 금지 문구 포함

## 카테고리 우선순위

1. `food`: 최우선. 방송 직후 검색 의도가 명확하고, 상호·지역·메뉴 기반 롱테일 대응이 쉽다. 단, 주소·영업시간·가격 교차 검증이 필수다.
2. `local`: 2순위. 지역 축제, 시장, 특산물, 여행 동선형 콘텐츠로 확장성이 있다. 단발 소개보다 지역 클러스터와 내부 링크를 만들어야 한다.
3. `life`: 3순위. 생활의 달인처럼 기술·인물·제품 관리법을 다룰 수 있으나, 상호성이 약한 회차는 검색 의도와 정보 이득을 먼저 확인한다.
4. `health`: 제한 운영. 생로병사의 비밀, 엄지의 제왕 등은 수요가 있지만 YMYL 리스크가 높다. 치료, 효능, 질환 개선 단정 없이 일반 정보·방송 요약·전문가 확인 권고 중심으로만 운영한다.

## 발행 속도 원칙

- 초기 2주는 하루 1~2개 초안 생산, 검증 통과분만 발행한다.
- 맛집/지역 글은 방송 후 24시간 내 초안, 48시간 내 검증 완료를 목표로 한다.
- 건강 글은 속도보다 안전을 우선하며, Researcher 검증 없이는 발행하지 않는다.
- 자동 발행 루틴은 최소 10개 수동 글의 검수 패턴이 쌓인 뒤 CEO 승인으로 전환한다.

## 콘텐츠 포맷 기준

### 맛집 글

- 제목: `프로그램명 오늘 방송 지역 상호 메뉴` 구조를 기본으로 한다.
- 필수 정보: 방송일, 회차/코너, 상호, 주소, 대표 메뉴, 방문 전 확인 사항, 출처.
- 본문: 방송 내용 직접 요약, 위치·메뉴·예약/휴무 확인 안내, 비슷한 지역 글 내부 링크.

### 지역 글

- 제목: `프로그램명 지역 시장/특산물/여행지` 구조를 기본으로 한다.
- 필수 정보: 지역, 장소, 접근 방법, 공식/지자체 출처, 방송 소개 포인트.
- 본문: 지역 맥락, 방문 전 확인 사항, 관련 맛집/특산물 연결.

### 생활 글

- 제목: `생활의 달인 분야 상호/인물 키워드` 또는 문제 해결형 키워드를 사용한다.
- 필수 정보: 방송 정보, 검증 가능한 업체/제품/기술 설명, 과장 금지 문구.
- 본문: 독자가 얻을 수 있는 체크리스트와 확인 기준 중심.

### 건강 글

- 제목: 질환 치료·효능 보장처럼 읽히지 않게 정보형으로 쓴다.
- 필수 정보: 방송 정보, 공신력 있는 일반 출처, 개인별 차이, 전문가 상담 권고.
- 금지: 완치, 치료, 예방 보장, 특정 제품 효능 단정, 복용량 지시.

## 검토 및 발행 기준

- Researcher는 확인된 사실과 불확실한 내용을 분리한다.
- Marketer는 검색 의도, 제목 후보, 메타 설명, 내부 링크 후보를 제공한다.
- Blogger는 복붙 없이 Markdown 원고를 작성하고 출처·방문 전 확인 사항을 포함한다.
- Coder는 frontmatter 스키마, 빌드, 커밋/푸시를 담당한다.
- QA는 발행 전 링크, 모바일 가독성, 메타 정보, 출처 표기를 확인한다.

## 제외 기준

- 공식 출처나 지도에서 상호·주소 확인이 불가능한 맛집 글
- 가격·영업시간을 방송 화면만 보고 단정해야 하는 글
- 건강기능식품, 치료법, 질환 개선 효과를 과장해야 클릭이 나오는 글
- 방송 대본이나 자막을 길게 옮겨야만 본문이 성립하는 글
- 성인, 도박, 불법 행위, 사기성 금융, 무허가 의료·법률 조언과 연결되는 주제

## 루틴 생성 전 운영안

1. Marketer가 1차 프로그램별 키워드·브리프 후보를 만든다.
2. Researcher가 우선순위 1~4 프로그램의 공식 페이지, 기사, 지도 확인 루트를 정리한다.
3. Blogger가 검증 완료 브리프 3건으로 초안 템플릿을 검증한다.
4. CMO가 초안 3건의 품질과 리스크를 검토한다.
5. CEO 승인 후에만 자동 모니터링/발행 루틴 설계를 CTO에게 넘긴다.

## 초기 실행 분해

- Marketer: 우선순위 1~4 프로그램의 키워드 맵과 글감 선별 기준 작성
- Researcher: 우선순위 1~4 프로그램의 출처 확인 플레이북 작성
- Blogger: 맛집/지역/생활/건강 원고 템플릿 초안 작성
- Coder/CTO: 루틴 또는 자동 발행이 필요하다는 CEO 승인 후에만 기술 설계 착수
