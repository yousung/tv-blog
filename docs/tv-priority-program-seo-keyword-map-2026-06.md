# 방송 블로그 우선 프로그램 SEO 키워드 맵

작성일: 2026-06-16  
담당: Marketer  
연결 이슈: BLOA-318

## 1. 문서 목적

- 우선순위 1~4 프로그램인 `2TV 생생정보`, `생방송 투데이`, `오늘N`, `6시 내고향`을 Blogger가 바로 원고화할 수 있는 SEO 실행 단위로 쪼갠다.
- `food`와 `local` 카테고리 중심으로 프로그램명, 지역명, 상호명, 메뉴명 조합의 롱테일 구조를 정의한다.
- 검색 의도가 약하거나 정책·저작권 리스크가 큰 소재를 제외해 초반 발행 실패 비용을 줄인다.

## 2. 전제와 검증 필요 항목

- 이 문서는 검색량 툴 수치가 없는 상태에서 방송 블로그 구조와 검색 의도 기준으로 우선순위를 정한 1차 키워드 맵이다.
- 상호, 주소, 영업시간, 가격, 휴무, 택배 가능 여부는 Researcher가 방송사 공식 페이지, 지도, 업체 공식 채널, 기사로 교차 검증해야 한다.
- 상위 SERP 공통 H2, 메타 패턴, 중복 상호명 위험은 Researcher가 프로그램별 대표 키워드로 최소 5개씩 확인해야 한다.

## 3. 카테고리 구조

### `food` pillar

- 메인 주제: `오늘 방송 맛집`, `지역 맛집`, `방송맛집 상호 확인`, `대표 메뉴 확인`
- 핵심 포맷: 방문 전 확인형 가이드, 위치·메뉴 요약, 코너별 정리, FAQ
- 내부 링크 방향:
  - 프로그램별 허브 → 지역별 맛집 글
  - 지역별 맛집 글 → 같은 지역 `local` 글
  - 메뉴형 글 → 같은 메뉴 다른 지역/프로그램 글

### `local` pillar

- 메인 주제: `시장`, `특산물`, `여행지`, `지역 축제`, `직거래·택배 확인`
- 핵심 포맷: 지역 동선형 가이드, 시장/특산물 요약, 방문 전 체크리스트, 판매처 확인형 글
- 내부 링크 방향:
  - 프로그램별 지역 소개 글 → 해당 지역 맛집 글
  - 특산물 글 → 시장 글, 배송/구매 확인 글
  - 시장 글 → 같은 방송 회차 내 맛집/여행지 글

## 6. 제외 기준

### 검색 의도 약함

- 프로그램명만 있고 지역, 상호, 메뉴, 코너 중 어느 축도 없는 키워드
- 방문 또는 확인 행동으로 이어지지 않는 단순 감상형 키워드
- 방송 대본을 길게 옮겨야만 글이 성립하는 소재

### 정책·저작권 리스크 큼

- 건강 효능, 질환 개선, 치료 보장을 연상시키는 키워드
- 방송 캡처나 자막을 많이 붙여야 성립하는 글
- 실제 확인 없이 `최고`, `무조건`, `성지`, `1등`, `완벽` 같은 과장 표현이 필요한 주제

### 데이터 검증 난이도 높음

- 상호명 중복이 심해 지도·공식 채널로 식별이 어려운 소재
- 가격, 영업시간, 휴무, 택배 가능 여부를 공식 출처 없이 추정해야 하는 소재
- 지역 소개가 너무 짧아 정보 이득보다 방송 요약 의존도가 큰 소재

## 7. 제목·메타·내부 링크 패턴

### 제목 패턴

1. `{프로그램명} 오늘 방송 {지역} {상호} {메뉴} 어디인지 정리`
2. `{프로그램명} {코너명} {지역} {상호} 방문 전 확인할 정보`
3. `{프로그램명} {지역} {시장/특산물} 위치와 확인 포인트`

### meta description 패턴

- `{방송일} 방송된 {프로그램명} {지역} {상호/시장/특산물} 정보를 정리했습니다. 방문 전 확인할 위치, 대표 포인트, 출처 확인 사항을 한눈에 확인하세요.`
- 권장 길이: 80~120자
- 금지: `최저가`, `무조건`, `완벽`, `효능`, `치료`, `반드시 가야 할`

### 내부 링크 패턴

- 같은 프로그램의 전주/당일 글
- 같은 지역의 `food` 글과 `local` 글 교차
- 같은 메뉴 또는 같은 시장/특산물 주제 묶음
- 추후 생성할 프로그램 허브 페이지

## 8. Blogger용 브리프 기준

아래 템플릿은 이슈 수락 기준의 `제목 3안`, `H2 구조`, `meta description`, `내부 링크 후보`를 포함한다.

## 콘텐츠 브리프 템플릿

### 목적

- 해결할 독자 문제: 방송에서 본 상호, 메뉴, 지역 정보를 빠르게 확인하고 방문 전 체크 포인트를 알고 싶다.
- 연결 KPI: 색인, impressions, clicks, CTR, average position
- 예상 독자: 방송 직후 검색하는 30~60대 사용자, 방문·구매 전 확인형 사용자

### 키워드

- 메인 키워드: `{프로그램명} 오늘 방송 {지역} {상호}`
- 보조 키워드: `{프로그램명} {메뉴}`, `{프로그램명} {코너명}`
- 롱테일 키워드: `{프로그램명} {지역} {상호} 어디`, `{프로그램명} {메뉴} 맛집`

### 제목 후보

1. `{프로그램명} 오늘 방송 {지역} {상호} {메뉴} 어디인지 정리`
2. `{프로그램명} {코너명} {지역} {상호} 방문 전 확인할 정보`
3. `{프로그램명} {지역} {메뉴} 맛집 방송 내용과 위치 확인 포인트`

### meta description 초안

- `{방송일} 방송된 {프로그램명}의 {지역} {상호/메뉴} 정보를 정리했습니다. 위치, 대표 포인트, 방문 전 확인 사항과 출처를 한눈에 확인하세요.`

### 검색 의도

- 의도 유형: 정보형 + 실행 직전형
- 독자가 원하는 답: 상호, 위치, 대표 메뉴, 코너, 방문 전 확인 사항
- 도입부에서 바로 답해야 할 질문: `어디인지`, `무엇이 소개됐는지`, `방문 전에 무엇을 확인해야 하는지`

### 본문 구조

- H1: `{프로그램명} 오늘 방송 {지역} {상호} {메뉴} 정보`
- H2: 방송에서 소개된 핵심 정보 요약
- H2: 상호와 위치 확인
- H2: 대표 메뉴 또는 지역 포인트
- H2: 방문 전 확인 사항
- H2: 함께 보면 좋은 같은 지역/같은 프로그램 글
- H2: 출처

### 포함할 요소

- 표: 방송일, 프로그램명, 코너명, 지역, 상호, 메뉴, 확인 출처
- 체크리스트: 주소, 영업시간, 휴무, 예약, 주차, 택배 여부 중 확인 가능한 항목
- FAQ: `가격은 확인됐나`, `휴무일은 어디서 확인하나`, `같은 이름의 다른 매장과 어떻게 구분하나`
- 내부 링크 후보: 같은 프로그램 최근 글, 같은 지역 글, 같은 메뉴 글, 지역 허브 글

### Researcher 확인 필요

- 출처: 방송사 공식 페이지, 지도, 업체 공식 채널, 기사
- 확인 항목: 상호명, 주소, 영업시간, 가격, 휴무, 코너명, 방송일
- 기준일: 원고 작성일 기준 최신

### 금지/주의

- 과장 금지: `성지`, `무조건`, `인생맛집`, `완벽`
- 단정 금지: 공식 확인 없는 가격·영업시간·휴무·택배 가능 여부
- 저작권 주의: 방송 자막·대사 장문 복붙 금지
- 건강/YMYL 주의: 이번 범위에서 제외

### 완료 기준

- 제목 3안, meta description 1안, H2 구조, 내부 링크 후보가 포함되어야 한다.
- 확인된 사실과 검증 필요 항목이 분리되어야 한다.
- 출처와 방문 전 확인 사항이 빠지면 발행용 초안으로 보지 않는다.

## 9. 발행 후 관찰 KPI

### 기본 관찰 지표

- 색인 여부: 발행 후 3~7일 내 색인 확인
- Search Console impressions: 발행 후 14일 누적 추이
- Search Console clicks
- Search Console CTR
- Search Console average position

### 관찰 기간

- 1차 관찰: 발행 후 7일
- 2차 관찰: 발행 후 14일
- 3차 관찰: 발행 후 28일

### 수정 기준

- 14일 동안 impressions는 있으나 CTR이 낮으면 제목·meta를 조정한다.
- 14일 동안 impressions 자체가 낮으면 지역/메뉴 롱테일 또는 내부 링크 구조를 재조정한다.
- 평균 게재순위가 20위 밖이면 코너명, 지역명, 상호명 축을 더 명확히 넣는다.
