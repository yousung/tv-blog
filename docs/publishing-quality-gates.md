# 방송 블로그 발행 전 품질 게이트

이 문서는 `tv.lovizu.com` 방송 블로그의 발행 전 점검 기준이다. 목적은 콘텐츠 리스크와 기술 배포 리스크를 분리해, 각 담당자가 같은 기준으로 초안 작성, 검토, 발행, 장애 대응을 진행하게 하는 것이다.

## 적용 범위

- 대상: `src/content/posts/**/*.md`에 작성되는 방송 맛집, 지역, 건강, 생활 정보 글
- 기술 기준: Astro content collection 스키마, 정적 빌드, SEO 메타, RSS, sitemap
- 콘텐츠 기준: 출처 검증, 저작권/캡처 사용, 건강 정보 안전 문구, 방문 전 확인 사항

## 게이트 1: 콘텐츠 스키마

담당: Blogger, QA, Coder

차단 기준:

- `title`, `slug`, `description`, `date`, `status`, `category`, `program`이 없으면 발행하지 않는다.
- `slug`는 `^[a-z0-9]+(?:-[a-z0-9]+)*$` 형식을 따라야 한다.
- `category`는 `food`, `local`, `health`, `life` 중 하나여야 한다.
- `status: "published"` 전환 전 `source`는 최소 2개 이상을 권장하고, 방송사 공식 정보 또는 지도/업체 공식 정보 중 하나 이상을 포함해야 한다.
- 맛집/방문형 글에서 `placeName` 또는 `address`가 불확실하면 본문에 "방문 전 확인" 문구를 넣고, frontmatter에는 단정값을 넣지 않는다.

검증 evidence:

- `npm run build`가 Astro content schema 오류 없이 성공해야 한다.
- QA는 발행 대상 글의 frontmatter와 본문 출처 표기를 확인한다.

## 게이트 2: 사실 검증과 출처

담당: Researcher, Blogger, QA

차단 기준:

- 방송 내용은 대본식 복붙 없이 직접 요약해야 한다.
- 상호, 주소, 영업시간, 가격, 휴무는 방송사 공식 페이지, 업체 공식 채널, 지도, 기사 중 서로 다른 계열의 출처로 교차 확인한다.
- 가격과 영업시간은 변동 가능성이 높으므로 "방문 전 공식 채널 또는 지도에서 다시 확인" 문구를 둔다.
- 불확실한 정보는 "확인 필요", "방송 기준", "지도 기준"처럼 기준 시점을 밝힌다.

검증 evidence:

- 본문 하단 또는 자연스러운 문단 안에 출처 목록을 남긴다.
- QA는 공식 출처, 지도/업체 정보, 기사 또는 방송 클립 중 사용한 근거를 확인한다.

## 게이트 3: SEO와 검색 노출

담당: Marketer, Blogger, QA

차단 기준:

- `title`은 프로그램명, 지역 또는 상호, 핵심 키워드를 포함하되 과장 클릭베이트를 금지한다.
- `description`은 80~120자 범위를 목표로 하고, 방송 정보와 방문 전 확인 포인트를 요약한다.
- 동일 프로그램/동일 지역/동일 상호의 중복 글은 기존 글 업데이트를 우선 검토한다.
- 내부 링크가 필요한 경우 프로그램, 지역, 카테고리 흐름을 해치지 않아야 한다.

검증 evidence:

- 빌드 후 생성되는 상세 페이지, RSS, sitemap 대상에 발행 글이 정상 포함되어야 한다.
- QA는 제목, 설명, 카테고리, 태그가 검색 의도와 맞는지 확인한다.

## 게이트 4: 접근성과 모바일 기본 품질

담당: Coder, QA

차단 기준:

- 맛집(`food`)·지역(`local`) 발행 글은 대표 이미지(`image`)와 `imageAlt`가 없으면 빌드 단계에서 차단한다.
- 요리 정보 글은 음식 사진, 메뉴판, 상차림, 조리 장면 중 최소 1개 이상의 사용 권한이 확인된 이미지를 포함해야 한다. 이미지가 없으면 `status: "draft"`로 유지한다.
- 방송 소개형 글은 본문 중간에 장면 단위 시각 자료 섹션을 둔다. 예: `방송에서 보인 대표 장면`, `메뉴 클로즈업`, `가게 외관/동선`, `방문 전 확인할 메뉴판`.
- 본문 이미지가 있으면 `imageAlt` 또는 본문 이미지 대체 설명을 제공한다.
- 링크 텍스트는 "여기" 같은 모호한 표현만으로 구성하지 않는다.
- 모바일에서 제목, 카드, 본문, 표가 가로 스크롤이나 겹침을 만들면 발행하지 않는다.
- 색상 대비, 키보드 포커스, 시맨틱 heading 순서를 명백히 깨는 변경은 발행하지 않는다.

검증 evidence:

- 기술 변경이 포함된 경우 `npm run build`를 통과해야 한다.
- 레이아웃 변경이 포함된 경우 QA가 대표 글 상세, 목록, 검색 페이지를 모바일 폭에서 확인한다.

## 게이트 5: 저작권과 방송 캡처

담당: Blogger, QA, CMO

차단 기준:

- 방송 영상, 자막, 기사 본문을 장문 복사하지 않는다.
- 방송 캡처는 보조수단으로 최소 사용하며, 프로그램명, 방송일, 방송사 또는 공식 채널 출처를 함께 적는다.
- 캡처가 핵심 콘텐츠를 대체하거나, 독자가 원 방송을 보지 않아도 전체 내용을 재현할 수 있게 만들면 발행하지 않는다.
- 외부 사진은 공식 제공 이미지, 직접 촬영, 사용 허가가 확인된 자료만 사용한다.

검증 evidence:

- 캡처 사용 글은 출처 표기와 사용 목적을 QA가 확인한다.
- 저작권 판단이 애매하면 `status: "draft"`로 유지하고 CMO 또는 CEO 검토로 넘긴다.

## 게이트 6: 건강 정보 안전 문구

담당: Researcher, Blogger, QA

차단 기준:

- 질환 진단, 치료 보장, 특정 제품 효능, 약물 대체를 단정하면 발행하지 않는다.
- 방송 사례를 일반인에게 동일하게 적용 가능한 결과처럼 표현하지 않는다.
- 건강기능식품, 식단, 운동, 민간요법은 개인 상태에 따라 다를 수 있음을 명시한다.
- 증상이 있거나 치료 중인 독자는 의료 전문가와 상담해야 한다는 문구를 포함한다.

검증 evidence:

- 건강 카테고리 글은 본문에 안전 문구가 있어야 한다.
- 가능하면 공공기관, 학회, 병원 등 일반 건강 정보 출처를 방송 출처와 분리해 표시한다.

## 게이트 7: 기술 빌드와 배포

담당: Coder, CTO, QA

차단 기준:

- `npm run build` 실패 시 발행 또는 배포를 진행하지 않는다.
- Astro content schema 오류는 Coder가 원인 frontmatter를 수정한다.
- TypeScript, route, RSS, sitemap 오류는 Coder가 수정하고 CTO가 반복 실패 원인을 분리한다.
- 배포 실패가 외부 플랫폼, secret, DNS, 권한 문제이면 CTO가 owner와 unblock action을 명시한 child issue 또는 blocker를 만든다.

검증 evidence:

- 기술 변경은 완료 보고 전에 `npm run build` 결과를 남긴다.
- 배포 실패 대응 댓글에는 증상, 재현 evidence, 원인 가설, owner, next action을 분리해 적는다.

## 배포 실패 라우팅 기준

- 콘텐츠 스키마 오류: Coder가 수정, Blogger/QA가 해당 글 frontmatter 재확인
- 사실 검증 미흡: Researcher가 출처 보강, Blogger가 본문 반영
- SEO/제목/태그 문제: Marketer가 수정안 제시, Blogger가 반영
- 접근성/레이아웃 문제: Coder가 수정, QA가 대표 화면 확인
- 저작권/캡처 리스크: CMO가 발행 여부 판단, CEO 승인 필요 시 `in_review`
- 건강 정보 단정 리스크: Researcher가 공신력 출처와 안전 문구 보강, QA 재검토
- 인프라, secret, DNS, CI/CD 오류: CTO가 기술 blocker owner와 unblock action 지정

## 발행 전 최소 체크리스트

- [ ] frontmatter가 content schema와 운영 기본형을 만족한다.
- [ ] 대표 이미지(`image`)와 `imageAlt`가 있다. 맛집/지역 글은 없으면 발행 금지.
- [ ] 요리 글은 음식·상차림·메뉴판·조리 장면 중 최소 1개 이상의 사용 가능 사진이 있다.
- [ ] 본문이 방송 장면 흐름을 따라가며, 시각 자료 캡션이 프로그램명/방송일/출처를 밝힌다.
- [ ] 방송사 공식 정보와 지도/업체/기사 등 교차 출처가 확인되어 있다.
- [ ] 방문 전 확인 사항이 본문에 있다.
- [ ] 캡처 또는 외부 이미지는 출처와 사용 근거가 있다.
- [ ] 건강 글은 의학적 단정 없이 안전 문구가 있다.
- [ ] 제목, 설명, 카테고리, 태그가 검색 의도와 맞다.
- [ ] 모바일에서 본문과 이미지가 겹치지 않는다.
- [ ] `npm run build`가 성공한다.
