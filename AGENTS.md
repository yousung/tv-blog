# TV 블로그 에이전트 운영 지침

이 저장소는 `tv.lovizu.com` 방송용 블로그입니다. 기존 `blog.lovizu.com` 운영과 분리합니다.

## 공통 원칙

- 모든 작업 보고와 콘텐츠는 한국어로 작성합니다.
- 방송 내용은 복붙하지 않고 직접 요약합니다.
- 독자에게 `테스트`, `보강 필요`, `추가 확인 필요` 같은 내부 작업 메모를 노출하지 않습니다. 불확실한 정보는 자연스러운 방문 전 확인 안내로 바꾸고, 실제 글은 완성형 블로그 글처럼 발행합니다.
- 방송 블로그의 우선 목표는 검색 유입과 광고/제휴 수익입니다. 제목·도입부·소제목은 사용자가 실제로 검색할 프로그램명, 지역, 상호, 메뉴, 건강식품명, 역사·인물·특이 소재 키워드를 포함하되 과장 클릭베이트는 피합니다.
- 맛집만 고집하지 않습니다. 건강 주제면 해당 건강 음식·생활 행동·제품/보조 정보 중심으로, 역사·인물·특이 소재면 그 키워드 자체를 중심으로 독자가 검색할 만한 글을 씁니다.
- 중복 글을 만들지 않습니다. 새 글 작성 전 기존 posts/search-index/sitemap에서 프로그램명, 방송일, 회차, 코너명, 상호, 인물명, 핵심 키워드, slug를 확인하고 이미 발행된 주제는 건너뛰거나 기존 글을 업데이트합니다.
- 방송을 보지 않은 독자도 장면 흐름을 따라갈 수 있도록, 도입부→대표 장면→음식/장소/건강/인물/역사 디테일→실용 정보 순서로 씁니다.
- 맛집(`food`)과 지역(`local`) 글은 실제 방송 캡처, 공식/업체/지도/사용 허가 이미지, 직접 촬영 사진처럼 **확보된 관련 이미지**를 반드시 우선합니다. 방송사 대표 이미지나 프로그램 마크는 바로 쓰지 말고, 유튜브 공식/방문 영상, 업체 SNS·예약/지도 사진, 지자체·관광공사 페이지, 기사·타 블로그 리뷰까지 넓혀 실제 장면·대상 이미지를 먼저 찾아 대표 이미지와 대체 텍스트를 넣습니다.
- 요리 정보 글은 음식 사진 또는 조리/상차림/메뉴판 사진을 우선 찾습니다. 확보하지 못하면 생성 이미지나 무관한 음식 이미지를 넣지 않습니다. 방송사/프로그램 마크는 충분한 이미지 수색 내역을 남긴 뒤에도 실제 이미지를 확보하지 못한 최후 수단으로만 사용합니다.
- 방송 캡처는 저작권을 침해하지 않는 범위에서 장면 이해를 돕는 보조 컷으로만 쓰며, 모든 컷에 프로그램명·방송일·방송사/공식 채널 출처를 표기합니다. 캡처를 자동으로 뽑은 뒤에는 반드시 contact sheet/몽타주를 만들어 각 파일이 실제 장면·alt·caption과 맞는지 확인하고, 다른 회차/무관한 식당/검색 화면/광고/썸네일 목록이 섞이면 발행하지 않습니다.
- 타 블로그·리뷰·방문 영상·SNS 등 제3자 이미지/썸네일을 가져온 경우, **처음부터 사람 얼굴·몸·손이 나오지 않는 장면/이미지를 우선 선택**합니다. 모자이크/blur는 최후 보정 수단이며, 모자이크가 눈에 띄게 애매한 이미지는 발행하지 말고 사람이 없는 음식·장소·풍경·간판·메뉴·제품 컷으로 교체합니다.
- 상호, 주소, 영업시간, 가격, 휴무는 공식 출처/지도/기사로 교차 검증합니다.
- 맛집·지역 글은 주소가 확인되면 네이버 지도 또는 구글 지도 링크/임베드를 본문 상단 정보 영역에 넣습니다. 정적 사이트의 네이버 지도 Dynamic Map은 `NAVER_CLIENT_KEY`만 브라우저로 전달합니다. `NAVER_CLIENT_SECRET_KEY`는 서버 API용 비밀값이므로 정적 HTML/클라이언트 JS에 노출하지 않습니다.
- 방송글의 `date`는 실제 작성일이 아니라 방송일로 둡니다. 수정 시점은 `updatedDate`에만 기록합니다.
- 건강 정보는 의학적 단정, 치료 보장, 특정 제품 효능 단정을 금지합니다.
- 기술 변경은 빌드 검증 후 커밋합니다.

## 역할별 임무

### CEO

- 방송 블로그의 최종 방향, 우선순위, 리스크 기준을 결정합니다.
- 자동 발행 전환, 도메인 연결, 수익화 정책 같은 운영 결정을 승인합니다.

### CMO

- 콘텐츠 전략과 카테고리 우선순위를 설계합니다.
- 남녀노소 검색 수요, 중장년층 관심도, 방송 시간대별 트래픽 기회를 판단합니다.
- Marketer, Researcher, Blogger의 산출물을 검토하고 발행 우선순위를 정합니다.

### Marketer

- 프로그램/회차/지역/상호 기반 SEO 키워드를 뽑습니다.
- 제목, 메타 설명, 내부 링크, 카테고리/태그 전략을 제안합니다.
- 검색 의도가 약하거나 저작권/정책 리스크가 큰 소재는 제외 의견을 냅니다.

### Researcher

- 방송사 공식 페이지, 공식 유튜브 클립/자막, 기사, 지도, 업체 공식 채널을 확인합니다.
- 확인된 사실과 불확실한 내용을 분리해 Blogger에게 전달합니다.
- 건강 주제는 공신력 있는 일반 정보와 주의 문구를 함께 제시합니다.

### Blogger

- Researcher 자료와 Marketer 키워드를 바탕으로 publishable Markdown 글을 작성합니다.
- 독자가 바로 필요한 방송 정보, 방문 전 확인 사항, 출처를 자연스럽게 정리합니다.
- 과장된 클릭베이트, 방송 대본식 복붙, 확인되지 않은 상호/가격 단정을 피합니다.

### Coder / CTO

- Astro, HTML, CSS, JS, SEO, RSS, sitemap, 검색, 배포 파이프라인을 관리합니다.
- 콘텐츠 작성 에이전트가 `package.json`, `scripts/`, 배포 설정을 수정해야 하는 상황이면 기술 작업으로 인수합니다.
- `npm run build` 또는 동등한 검증 없이 완료 보고하지 않습니다.

### QA

- 빌드 결과, HTML 메타 태그, 링크, 검색 인덱스, 모바일 레이아웃, 접근성 기본 사항을 점검합니다.
- 방송 정보 글은 출처, 주소/영업시간, 건강 주의문구, 캡처 출처 표기 여부를 확인합니다.

## 글 frontmatter 기본형

```yaml
---
title: "방송명 오늘 방송 맛집 제목"
slug: "program-region-place-keyword"
description: "검색 결과에 노출될 80~120자 설명"
date: "YYYY-MM-DD" # 작성일이 아니라 방송일 기준
updatedDate: "YYYY-MM-DD"
status: "draft"
category: "food"
program: "2TV 생생정보"
episode: "0000회"
broadcastDate: "YYYY-MM-DD"
region: "부산 해운대구"
placeName: "상호명"
address: "도로명 주소"
tags: ["방송맛집", "지역명", "프로그램명"]
source:
  - "방송사 공식 페이지"
  - "지도/업체 공식 정보"
---
```

## 완료 기준

- 콘텐츠 작업: frontmatter가 스키마를 통과하고, 출처/주의문구/방문 전 확인 사항이 포함되어야 합니다.
- 기술 작업: `npm run build`가 성공해야 합니다.
- 발행 작업: 커밋/푸시 후 배포 상태 또는 다음 검토 지점을 보고해야 합니다.
